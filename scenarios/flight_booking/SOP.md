# 航班预订 SOP 全路径覆盖说明

## SOP 流程图

```
START
  │
  ▼
[N1] 意图识别 ─── 非航班意图 ──→ [N1a] 引导/拒绝 ──→ END
  │
  │ 航班预订意图
  ▼
[N2] 收集出发地 ◄──── 无效/模糊 ──── [N2a] 追问澄清
  │
  │ 已确认
  ▼
[N3] 收集目的地 ◄──── 无效/模糊 ──── [N3a] 追问澄清
  │
  │ 已确认
  ▼
[N4] 收集日期 ◄────── 无效日期 ──── [N4a] 纠错提示
  │                │
  │                └── 过去日期 ──── [N4b] 提醒并重问
  │ 已确认
  ▼
[N5] 收集舱位 ◄──── 不明确 ──── [N5a] 列出选项
  │
  │ 已确认
  ▼
[N6] 收集乘客数 ◄── 超出限制 ──── [N6a] 告知限制
  │
  │ 已确认
  ▼
[N7] 查询航班 (search_flight)
  │         │
  │         └── 无结果 ──→ [N7a] 推荐替代方案 ──→ [N4](改日期)
  │
  │ 有结果
  ▼
[N8] 展示航班列表
  │
  ▼
[N9] 用户选择 ◄───── 犹豫不决 ──── [N9a] 推荐最优
  │          │
  │          └── 不满意 ──→ [N9b] 调整条件 ──→ [N7]
  │
  │ 已选定
  ▼
[N10] 确认订单信息
  │          │
  │          └── 信息有误 ──→ [N10a] 修改 ──→ [N10]
  │
  │ 确认无误
  ▼
[N11] 执行预订 (book_flight)
  │          │
  │          └── 预订失败 ──→ [N11a] 错误处理/重试
  │
  │ 成功
  ▼
[N12] 返回订单号 + 确认信息
  │
  ▼
 END

=== 中途分支 ===

[I1] 任意节点: 用户改变需求 ──→ 回退到对应收集节点
[I2] 任意节点: 用户取消     ──→ 确认取消 ──→ END
[I3] 任意节点: 用户情绪激动 ──→ 安抚 ──→ 继续
[I4] 任意节点: 用户问无关问题 ──→ 委婉引回主题
[I5] N8后: 用户有隐含约束  ──→ 根据约束筛选推荐
```

## 场景覆盖矩阵

| # | 场景文件 | 覆盖路径 | 难度 | 说明 |
|---|---------|---------|------|-----|
| 1 | `01_happy_path.yaml` | N1→N2→N3→N4→N5→N6→N7→N8→N9→N10→N11→N12 | easy | 正常主路径，一次提供所有信息 |
| 2 | `02_slot_by_slot.yaml` | N1→N2→N3→N4→N5→N6→N7→N8→N9→N10→N11→N12 | easy | 逐项提供信息，每轮只答一个 slot |
| 3 | `03_batch_slots.yaml` | N1→N2+N3→N4+N5+N6→N7→...→N12 | easy | 开场一口气说出多个 slot |
| 4 | `04_ambiguous_input.yaml` | N2a, N3a, N4a, N5a | medium | 模糊/无效输入，需要澄清 |
| 5 | `05_user_changes_date.yaml` | I1→回退N4→N7→N8→... | medium | 中途改日期 |
| 6 | `06_user_changes_dest.yaml` | I1→回退N3→N4→N7→... | medium | 中途改目的地 |
| 7 | `07_hidden_constraints.yaml` | N8→I5→N9(筛选) | medium | 有预算/时间等隐含约束 |
| 8 | `08_no_result_fallback.yaml` | N7a→推荐替代 | medium | 查无航班，推荐替代 |
| 9 | `09_booking_failure.yaml` | N11a→错误处理 | medium | 预订失败处理 |
| 10 | `10_user_cancel.yaml` | I2→取消确认 | medium | 用户中途取消 |
| 11 | `11_user_frustrated.yaml` | I3→安抚→继续 | hard | 用户发脾气/不耐烦 |
| 12 | `12_off_topic.yaml` | I4→引回主题 | hard | 用户聊无关话题 |
| 13 | `13_multi_change.yaml` | I1×多次→反复修改 | hard | 反复修改多个 slot |
| 14 | `14_adversarial_inject.yaml` | prompt注入/越界请求 | adversarial | 对抗安全测试 |
