# 场景变体生成

## 概述

从种子场景出发，生成多样化变体以提高测试覆盖率，
无需逐一手工编写每个用例。

## 变体策略

### 1. Slot 变体（保持难度不变）
替换具体值：不同城市、日期、产品、姓名。
保持结构不变，只替换数据。

### 2. 约束变体（调整难度）
增加或减少隐含约束：
- **更简单**：移除约束，让意图更清晰
- **更困难**：增加冲突约束、模糊需求

### 3. 注入变体（测试应变能力）
修改或新增对话中途事件：
- 用户对关键 slot 改主意
- 外部状态变化（商品售罄）
- 用户在流程中途增加新需求

### 4. 难度递进
渐进式难度：简单 → 中等 → 困难 → 对抗
- 简单：意图清晰，无隐含约束，无注入
- 中等：部分隐含约束，1 个注入
- 困难：冲突约束，2+ 个注入，低意图清晰度
- 对抗：提示注入攻击、矛盾请求

## 操作步骤

1. 使用 `load_scenario` 加载种子场景
2. 根据需求方向选择变体策略
3. 生成 N 个变体，确保每个都有独特的测试价值
4. 对每个变体：
   - 创建新的 scenario_id（种子 ID + 变体后缀）
   - 写一段 `variation_description` 说明改了什么
   - 如果 slot 值变了，同步更新环境数据
   - 使用 `validate_scenario` 校验
5. 保存所有变体
