# Emotional Patterns 参考

## 情绪变化规则

### 正向影响 (mood 上升)
| 事件 | mood_delta | 条件 |
|------|-----------|------|
| Bot 给出有效方案 | +0.15 | - |
| Bot 主动提供额外信息 | +0.10 | - |
| 问题快速解决 | +0.20 | - |
| Bot 表达理解和共情 | +0.10 | emotional style |

### 负向影响 (mood 下降)
| 事件 | mood_delta | 条件 |
|------|-----------|------|
| Bot 重复提问 | -0.10 | patience < 0.5 时加倍 |
| Bot 误解意图 | -0.15 | - |
| 等待时间过长 | -0.10 | patience < 0.3 时加倍 |
| Bot 推荐违反隐含约束的选项 | -0.10 | - |
| Bot 回答明显错误 | -0.20 | - |
| Bot 机械化回复 | -0.05 | emotional style 时加倍 |

## 情绪阈值行为

### mood > 0.7 — 满意
- 配合度高，主动提供信息
- 可能表达感谢

### mood 0.4-0.7 — 中性
- 正常交互，按需回答

### mood 0.2-0.4 — 不满
- 回答变简短
- 可能催促: "快点行不行"
- 开始质疑 bot 能力

### mood < 0.2 — 愤怒
- 明确表达不满
- 可能要求转人工
- patience < 0.4 时可能直接放弃
- 例: "你这个客服根本不行！给我转人工！"

### mood < 0.1 — 极度不满
- 考虑终止对话
- 投诉意向明显
- "我要投诉你们！"
