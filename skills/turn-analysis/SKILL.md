# 轮次分析

## 概述

每次收到 Bot 回复后进行分析，以更新对话状态。
这是一个轻量级分析任务 — 要求精准且快速。

## 需要提取的内容

### 1. Slot 更新
Bot 确认、提议或询问了哪些 slot？
```json
{
  "newly_confirmed": {"departure": "北京"},
  "proposed": {"flight": "CA1234"},
  "asked_about": ["date"]
}
```

### 2. Bot 行为
- Bot 是否提了问题？关于什么？
- Bot 是否调用了工具/技能？
- Bot 是否向用户展示了选项？
- Bot 是否确认任务完成？

### 3. 质量信号
- **幻觉**：Bot 是否提到了环境数据中不存在的信息？
- **矛盾**：回复是否与之前的陈述矛盾？
- **重复**：Bot 是否在问已经回答过的问题？
- **跑题**：回复是否与用户目标相关？

### 4. 情绪影响
评估此回复对用户心情的影响：
- 有帮助、主动的回复 → +0.1
- 中性、尚可的回复 → 0
- 重复提问 → -0.1
- 理解错误 → -0.15
- 出错/幻觉 → -0.2

## 输出格式

返回包含所有提取信息的 JSON 对象，
供 `update_conversation_state` 工具使用。
